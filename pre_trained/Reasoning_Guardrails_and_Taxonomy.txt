# Reasoning guardrails and explicit taxonomy facts

## Tool categories
- JMeter: performance/load/stress testing tool.
- Bugzilla: defect/issue tracking system.
- Selenium/Cypress: UI automation testing tools.
- Postman/RestAssured: API testing tools.

## Static vs dynamic analysis
- Static analysis inspects source code or artifacts without executing the program.
- Dynamic analysis requires runtime execution and observation.
- If a task claims static analysis during runtime execution, that claim is incorrect.

## Automation criteria
- Best automation candidates: stable, repeatable, high-frequency regression checks.
- Poor automation candidates: highly volatile scenarios, one-off exploratory checks.
- Automation should reduce repetitive effort, not replace all critical human judgment.

## Option evaluation rules
1. Evaluate each option exactly once in order.
2. Do not loop or re-evaluate the same option repeatedly.
3. "All variants" is valid only if every option is individually validated as correct.
4. If any option is incorrect, reject "all variants".
